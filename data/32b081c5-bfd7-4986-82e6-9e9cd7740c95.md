On October 25, 2022, Cloudflare experienced a partial outage affecting its Tiered Cache system. A software release containing a bug began rolling out at 08:40 UTC, leading to an increase in HTTP 530 errors. The issue escalated as the release spread globally, with peak impact observed at 17:28 UTC. A rollback was initiated and confirmed at 17:03 UTC, with the impact ending once the rollback was complete in all upper tier data centers by 18:04 UTC.

The incident caused approximately 5% of all HTTP requests to fail at peak, returning a 530 status code. This affected users with Tiered Cache, Cloudflare Images, or Bandwidth Alliance configured, as requests requiring an origin fetch through the tiered cache hierarchy were impacted. Content served directly from a data center's local cache remained unaffected.

The root cause was identified as new distributed tracing code added to the Tiered Cache logic. This tracing code inadvertently cleared control headers, which are essential for routing requests and internal DNS lookups for origin server IP addresses. Consequently, critical hostname information was missing, leading to the 530 DNS errors.

Immediate remediation involved rolling back the faulty software release. To prevent recurrence, Cloudflare plans to include larger Tiered Cache upper tier data centers in earlier release stages, expand acceptance test coverage for various Tiered Cache topologies, implement more aggressive alerting for requests lacking full context, and ensure systems fail fast during development and testing.
