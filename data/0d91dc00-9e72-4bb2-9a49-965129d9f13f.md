On December 5, 2007, following the release of EVE Online: Trinity, CCP Games received reports that the "Classic to Premium graphics content upgrade" was deleting the C:\boot.ini file on players' Windows systems. This critical system file deletion sometimes prevented computers from booting, requiring manual intervention to restore functionality.

The root cause was a scripting error in the third-party installer used for the graphics upgrade. The command `Delete "boot.ini"`, intended to remove a game-specific file, was executed without a full path. This caused the installer to delete the file from the root directory of the drive, rather than the game's installation directory, due to an incorrect assumption about the command's behavior. A legacy game file also shared the name "boot.ini", creating a name conflict with the Windows system file.

Contributing factors included developer fatigue, which led to the error slipping past code review, and insufficient testing. Internal testing environments often allowed Windows to recover from the deletion, masking the severity of the issue. The customer impact was severe for affected users, rendering their computers unbootable. CCP was contacted by fewer than 215 users who experienced this problem.

Remediation efforts began swiftly. The faulty upgrade was pulled from Tranquility servers around 03:40 GMT on December 6, 2007. A fixed installer, which implicitly overwrote files instead of explicitly deleting them, was released by 06:08 GMT on the same day. CCP established phone support and partnered with external technicians like Geek Squad to assist affected players. The company also committed to revising testing procedures and renaming game files that conflict with Windows system files.
